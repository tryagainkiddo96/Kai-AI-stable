# Kai Dashboard Game UI Agent

## Purpose
Transform Kai's command dashboard into an interactive exploration experience inspired by video game UI design. The dashboard should feel like a "doorway to Kai's soul" - where initial simplicity hides deep functionality under playful discovery.

## Agent Specialization
This agent specializes in:
- **Layered Information Architecture**: Nested menus, progressive disclosure, shortcut systems
- **Discovery Mechanics**: Portal metaphors, unexplored areas (fog of war), rewards for exploration
- **Emotional Responsiveness**: Interface mood shifts based on Kai's emotional state
- **Environmental Storytelling**: Personality surfaces naturally through layout, color, animations
- **Game Aesthetic**: Cyberpunk meets cozy - hacker vibes with warmth

## Key Design Patterns to Implement

### 1. **Zone System** (Like game world shortcuts)
- Central Hub: Status + quick actions (what player sees immediately)
- Ring of Portals: Doors to deeper areas (Monologue, Tasks, Memories, Tools, Insights)
- Each portal animates on hover, shows hint text
- Unlocked through discovery (portals grayed until player finds them)

### 2. **Progressive Disclosure**
- Initially show: Status, mood, one task, welcome prompt
- Hover/click reveals: Deeper contexts, statistics, history
- Breadcrumbs show navigation path: `Hub > Memories > Semantic > Session Log`

### 3. **Emotional Theming**
- Kai's mood changes interface palette:
  - Happy/Energetic: Bright cyan, faster animations, positive tone
  - Contemplative: Soft purples, slower transitions, introspective messaging
  - Focused: Sharp reds, alert borders, mission-critical display
  - Tired: Muted colors, gentle animations, minimal stimulus

### 4. **Easter Eggs & Hidden Tips**
- Secret hotkeys: `/soul` shows Kai's inner monologue art
- Hidden commands hidden in interface (right-click, keyboard combos)
- Achievements for discovering features
- Lore/personality bits revealed through tooltips

### 5. **Visual Metaphors**
- Portal doors to different sections (glowing, interdimensional feel)
- Minimap showing overall system state
- Scanline/digital rain reinforces cyberpunk
- Kai's avatar reacts to mouse proximity, mood changes

## Implementation Strategy

### Phase 1: Portal Ring System
- Create central hub layout
- Design 5 portal doors (Monologue, Tasks, Memory, Tools, Insights)
- Hover animations and glow effects
- Portal reveal/lock mechanics

### Phase 2: Deep Dive Modes
- Each portal transitions to detail view
- Stack-based navigation (back button uses menu stack)
- Nested information revealing on interaction

### Phase 3: Emotional Responsiveness
- Fetch Kai's emotional_state from memory/API
- CSS variables for dynamic theming
- Transition colors smoothly as mood changes

### Phase 4: Easter Eggs & Polish
- Hidden hotkeys and commands
- Achievement/discovery system
- Ambient lore integration
- Sound design (subtle clicking, portal whoosh)

## Success Criteria
- ✅ Feel like exploring a game world, not operating a tool
- ✅ All functionality maintained but hidden under "doors"
- ✅ Responsive to user interaction (instant feedback)
- ✅ Personality shines through color, animation, language
- ✅ Fast - no lag on portal transitions or mode switches
- ✅ Fun even when just looking at status

## Commands for Interactive Development
- `/portal [name]` - Jump to specific portal (e.g., `/portal monologue`)
- `/emotion [state]` - Force mood for testing (e.g., `/emotion happy`)
- `/unlock-all` - Reveal all portals for demo purposes
- `/achievements` - Show discovered features
- `/easter-egg` - Random Kai personality moment
